## 2026-03-23 - Upgrade MatrixScanner to extract internal file symbols
**Anomaly:** Full DAG tree generation without internal symbol mapping inside .rs files restricted the spatial topology payload to just files and dependencies. Using AST crates like `syn` would violate the zero-copy and self-hosting constraints.
**Resolution:** Implemented a lightweight, zero-copy lexical line parser in `scan_file` controlled by a `ScanDepth` parameter (Interface vs. DeepAST). Added an `is_single_file` boundary to strictly enforce that symbol extraction only happens on single file targets to prevent DAG payload detonation during root workspace scans. Symbols are mapped as flat children under the file's node in `topology_edges`.
