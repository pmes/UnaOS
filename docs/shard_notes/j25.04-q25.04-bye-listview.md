## 2026-03-29 - The ListView Purge
**Anomaly:** GTK4's ListView recycling engine failed to handle dynamically sized chat bubbles, resulting in state-bleed, violent scroll jitter, and a bloated Frankenstein widget skeleton that fought the layout engine.
**Resolution:** Scrapped the ListStore/ListView paradigm entirely. Implemented a direct gtk4::Box appending strategy with a strict 100-node DOM limit culler and vadjustment-based lazy loading to maintain an ultra-light UI tree.
