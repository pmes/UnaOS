## 2026-03-30 - [Architectural Audit & Blueprint: Rebuilding macOS Native UI from J02.02]
 **Structural Anomalies Found:** J02.02 failed catastrophically due to mixing Objective-C block memory management (via `block2::RcBlock`) with standard `dispatch_async` routing in `spline.rs`, resulting in memory leak traps. Further, J02.02 violated the `objc2` v0.6.4 API syntax boundaries, failed to explicitly mark delegates with `MainThreadOnly` (via inheriting `NSResponder`), failed to implement `NSObjectProtocol` for custom classes, incorrectly used mutable struct `define_class!` definitions rather than unit structs with `Ivars`, and attempted to rely on GTK layout terminology rather than native `NSLayoutConstraint` arrays with `translatesAutoresizingMaskIntoConstraints = NO`.

 **The Woz Blueprint:** Apply strict Can-Am rules to rebuild `libs/quartzite/src/platforms/macos`. Replace all `block2` usage in `spline.rs` cross-thread dispatch boundaries with `dispatch2::DispatchQueue::main().exec_async()`. Subclass all custom app/workspace delegates from `NSResponder` instead of `NSObject` to properly inherit `MainThreadOnly` requirements. Use explicit `Ivars` attached to unit structs via `#[ivars = StructIvars]` for storing `Retained<T>` references wrapped in `RefCell`s to tie UI component lifecycles properly to the delegate objects. Use `objc2::sel::Sel` for selectors, `msg_send!`, and explicit `NSArray::from_slice` for fully wired programmatic `NSLayoutConstraint` assemblies spanning the split view (`sidebar.rs` and `comms.rs`).
