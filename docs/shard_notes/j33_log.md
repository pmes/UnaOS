## 2024-05-24 - [Photon Ignition: Zero-Copy ARW Architecture]
**Anomaly:** Full decompression of Sony ARW files inherently challenges strict zero-copy guidelines because Bayer tiles must be reconstructed into contiguous arrays before math can be applied without causing catastrophic CPU cache misses.
**Resolution:** Can-Am Rules applied. Uncompressed files (Compression=1) are mapped and iterated directly from the `&[u8]` slice into the Bayer intermediate buffer. We authorize the allocation of an intermediate `Vec<u16>` buffer *only* for the unpacked Bayer array. Rayon is utilized to rapidly demosaic this intermediate buffer into the absolute final `Vec<f32>` RgbBuffer. All TIFF IFD traversal is entirely zero-copy using slice references. No allocations occur until we hit the raw image data payload.
