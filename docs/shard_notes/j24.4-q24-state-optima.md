## 2024-03-22 - [J24.4 "Optima" State Optimization] -> **Anomaly:** GTK was instantiating its own `WorkspaceTetra`, creating a dual-state reality. The Qt Embassy was blank due to a failed boot sync. Furthermore, the `matrix` handler contained legacy GTK UI code from the deprecated `una` IDE app.
-> **Resolution:** Purged GTK from `matrix`. Implemented a pure `TreeNode` DAG builder (`build_genesis_tree`) in the `matrix` domain. Orchestrator now builds the master state once and pipes it immutably to both GTK (`sidebar.rs`) and Qt (`vein_bridge.rs`).

**Blast Radius:** `libs/quartzite/src/tetra.rs` was completely deleted to enforce pure domain boundary. This broke other UI modules referencing `tetra`, but the core pipeline for Matrix -> Orchestrator -> GTK/Qt Matrix bindings was properly transitioned to use the newly defined `bandy::state` primitives (`TopologyNode`, `TopologyState`, `WorkspaceState`, `ViewEntity`). A follow-up Shard will sweep and fix the remaining compilation errors across the wider UI suite.
