## 2024-04-03 - Resolving Squeezed Layouts and Tree States natively with Objc2
**Anomaly:** Used `objc2::msg_send_id!` to retrieve ObjC objects like `widthAnchor` and `userInfo`, triggering deprecated macro warnings because modern `msg_send!` natively handles `Retained<T>` ARC pointer conversions automatically.
**Resolution:** Explicitly replaced `objc2::msg_send_id!` with the standard `msg_send!` macro across layout constraint initialization and `NSNotification` user info retrievals. Natively anchored `widthAnchor` constraints with `constraintGreaterThanOrEqualToConstant:` to fix layout squishing and utilized `outlineViewItemDidExpand:` / `outlineViewItemDidCollapse:` via the NSOutlineViewDelegate to natively sync visual expansion state with the Rust Backend's `RefCell<bool>`.
