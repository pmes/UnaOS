## 2024-04-03 - Resolving Squeezed Layouts and Tree States natively with Objc2
**Anomaly:** Used `msg_send!` to retrieve ObjC objects like `widthAnchor` and `userInfo` assigned to `Retained<T>`, leading to failing trait bounds compilation errors (`Retained<T>: Encode is not satisfied`) and severe theoretical memory corruption (bypassing ARC retains).
**Resolution:** Explicitly replaced `msg_send!` with `objc2::msg_send_id!` when calling objective-c methods that return Objective-C objects into `Retained<T>`. Natively anchored `widthAnchor` constraints with `constraintGreaterThanOrEqualToConstant:` to fix layout squishing and utilized `outlineViewItemDidExpand:` / `outlineViewItemDidCollapse:` via the NSOutlineViewDelegate to natively sync visual expansion state with the Rust Backend's `RefCell<bool>`.
