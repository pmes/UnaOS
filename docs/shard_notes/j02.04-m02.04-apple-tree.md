## 2024-03-31 - [Architectural Audit & Blueprint: AppKit Bridge Compliance]

**Anomaly:** Previous shards failed to compile the macOS UI bridge because they failed to respect strict Objective-C macro syntax inside `objc2` v0.6.4 and completely misunderstood the native layout requirements (e.g., using integer constraints or trying to write layout stubs with GTK-styled structs).

**Resolution:** Executed the Can-Am solution with a pure, unapologetic AppKit approach:
1.  **Strict Modularity:** Implemented the Right Pane as `comms.rs` (not `reactor.rs`) and segregated `sidebar.rs`, `toolbar.rs`, and `spline.rs` within `libs/quartzite/src/platforms/macos`.
2.  **Explicit Boilerplate:** Completely abandoned auto-resizing masks for pure programmatic `NSLayoutConstraint` code wrappers via `NSArray` slices and the explicit relations (`NSLayoutAttribute` / `NSLayoutRelation`).
3.  **Undocumented Protocols:** Supplied empty `unsafe impl` declarations for AppKit super-protocols like `NSTextDelegate` and `NSControlTextEditingDelegate` which are not implemented by default when generating custom subclasses.
4.  **Async Thread Exits:** Purged `block2` closures and strictly utilized `dispatch2::DispatchQueue::main().exec_async` to cross back over into the main UI thread during `SMessage` reception.
5.  **RefCell Memory Architecture:** Attached strong `Retained<T>` references utilizing `RefCell` to store component lifetimes strictly inside `#[ivars = DelegateIvars]`. This explicitly ties the AppKit components (e.g. `NSToolbar`, `NSOutlineView`, `NSTextView`) to the instantiated delegate.
