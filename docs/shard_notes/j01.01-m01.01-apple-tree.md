## 2026-05-18 - [Architectural Audit & Blueprint]

**Structural Anomalies Found:**

1.  **Cargo.toml Feature Deficits:** The current `Cargo.toml` targets for `objc2-app-kit` lack the specific feature flags required to access `NSLayoutConstraint`, `NSSplitViewController`, `NSOutlineView`, and Grand Central Dispatch (GCD) blocks.
2.  **AppKit `NSApplicationDelegate` Bootstrapping:** Bridging Rust's `FnOnce(&NativeWindow) -> NativeView + 'static` into an Objective-C delegate requires careful interior mutability (e.g., `RefCell<Option<F>>`) inside a custom `declare_class!` instance to ensure the closure is only consumed once upon `applicationDidFinishLaunching:`, while satisfying Objective-C's instance variable requirements (which often demand `Send`/`Sync` or main-thread-only wrappers).
3.  **Cross-Thread Event Routing (`spline.rs`):** We cannot map `async_channel::Receiver<SMessage>` directly to AppKit UI mutations without explicitly dispatching back to the main thread. AppKit will hard-crash if updated from a background thread.

**The Woz Blueprint:**

### Phase 2 Execution Plan - macOS Native UI Bridge (`libs/quartzite/src/platforms/macos`)

**Step 1: Cargo.toml Payload Augmentation**
*   Inject the missing `objc2-app-kit` features: `NSLayoutConstraint`, `NSSplitViewController`, `NSSplitViewItem`, `NSOutlineView`, `NSTableColumn`, and any required text layout delegates.
*   Ensure `block2` and GCD/`libdispatch` crates or appropriate `objc2-foundation` features (like `NSThread::mainThread()`) are available for cross-thread messaging. (No new dependencies outside of authorized `objc2` ecosystem additions).

**Step 2: Core Module Scaffolding (`mod.rs`)**
*   Define `Backend { app: Retained<NSApplication> }`.
*   Implement `Backend::new<F>`:
    *   Acquire `[NSApplication sharedApplication]`.
    *   Define a custom `NSApplicationDelegate` via `declare_class!`.
    *   Store the `bootstrap_fn` safely inside the delegate (using `std::cell::RefCell<Option<Box<dyn FnOnce(...)>>>` mapped to a custom ivar or associated object).
    *   In `applicationDidFinishLaunching:`, construct the root `NSWindow` (alloc/init with `NSWindowStyleMaskTitled | NSWindowStyleMaskClosable | NSWindowStyleMaskResizable`).
    *   Consume the `bootstrap_fn`, passing the window reference, and receive the root `NSView`.
    *   Set the window's `contentView` to the returned `NSView`.
    *   Call `[window makeKeyAndOrderFront:nil]`.
*   Implement `Backend::run`:
    *   Invoke `[self.app run]`.

**Step 3: The Verbosity Mandate - UI Construction (e.g., `widgets.rs` or directly in `bootstrap_fn`)**
*   **No Autoresizing Masks:** Every `NSView` constructed must have `setTranslatesAutoresizingMaskIntoConstraints:NO`.
*   **Manual Layout Constraints:** Build extensive arrays of `NSLayoutConstraint`. Example:
    ```rust
    // Concept:
    let constraints = NSArray::from_vec(vec![
        child_view.widthAnchor().constraintEqualToAnchor(parent.widthAnchor()),
        // ... etc ...
    ]);
    NSLayoutConstraint::activateConstraints(&constraints);
    ```
*   **Complex Views (`NSSplitViewController` & `NSOutlineView`):**
    *   Use `declare_class!` to implement `NSOutlineViewDataSource` and `NSOutlineViewDelegate`.
    *   Manually assemble the `NSSplitViewItems` and add them to the `NSSplitViewController`.

**Step 4: The Spline Translator (`spline.rs`)**
*   Establish the background listener for `async_channel::Receiver<SMessage>`.
*   When a message arrives (e.g., `SMessage::StateInvalidated`), it must not directly touch AppKit.
*   Use `dispatch_async(dispatch_get_main_queue(), block)` (via `block2::RcBlock::new`) to execute the UI mutation closure safely on the main thread.
*   Ensure the Spline module strictly maps abstract OS concepts to granular `NSOutlineView::reloadItem:` or `NSTextView::setString:` calls, shielding the rest of UnaOS from AppKit semantics.

**End of Blueprint.** Awaiting Phase 2 Authorization.
