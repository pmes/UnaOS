## 2026-03-31 - [Architectural Audit & Blueprint]

**Structural Anomalies Found:**
Passing `MutexGuard` across `tokio::spawn` await boundaries using `Arc::new(self.inner.lock().unwrap())` violates Rust's `!Send` constraints and guarantees compilation failure. Failing to cast void `msg_send!` returns to `()` strictly violates the `objc2` v0.6.4 type safety rules. Unanchored delegates drop immediately, leaving dangling pointers or missing UI behavior.

**The Woz Blueprint:**
- State wrapped in an `Arc<Mutex<T>>` *inside* the class definition and `Arc::clone`'d into the Tokio spawn task avoids locking over await boundaries.
- Empty `NSArray` types strictly require `NSArray::new()` over `NSArray::from_retained_slice(&[])` when inferring generic type arguments.
- Extracted and anchored `toolbar_delegate` across `create_window` to keep `NSToolbar` strictly referenced in `AppDelegateIvars`.
