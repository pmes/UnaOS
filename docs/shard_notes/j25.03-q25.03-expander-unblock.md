## 2026-03-28 - Bypassing ListView items_changed Optimization for Zero-Copy State Alignment
**Anomaly:** The `left_expand_btn` and `right_expand_btn` in `comms.rs` depressed but failed to toggle the message expansion. GTK4's `gio::ListStore::items_changed(pos, 1, 1)` optimization was ignoring the signal because the underlying `HistoryObject` pointer in the store remained identical, and manual UI bindings in `connect_bind` lack `notify` property listeners.
**Resolution:** Extracted the truncation and DOM-mutation logic into a pure Can-Am helper function (`apply_expansion_state`). The click handlers in `connect_setup` now use `unsafe { item.data("widgets") }` to retrieve the `BubbleWidgets` struct and invoke the helper directly. This surgically mutates the view state synchronously, bypassing the `ListView`'s redraw cycle entirely without splicing the data store.

## 2026-03-28 - Enforcing Responsive Layout Stagger in Chat View
**Anomaly:** Chat bubbles failed to visually stagger on wide screens, sitting directly over each other. The `msg_label` natural width request (`85 chars`) was consuming nearly all available window space, preventing `halign` from creating a visible left/right flow.
**Resolution:** Reduced `max_width_chars` to `65` to cap the natural expansion earlier. Injected asymmetric margins (`margin_start(64)` for Architect, `margin_end(64)` for AI) directly into the `connect_bind` state. This guarantees a physical UI boundary that forces a visually distinct, balanced stagger as the window expands, while naturally compressing on narrow constraints.
