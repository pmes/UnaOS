## 2024-05-24 - J25.01 "Loom" 🧵 :: UI Physics and Gneiss Abstraction
**Anomaly:** The TeleHUD GTK4 `ListItem` selection was failing due to inner widgets swallowing pointer events. The matrix navigation tree scrolled erratically when the dataset was spliced. User chat bubbles were consuming all horizontal space instead of right-justifying, and the Network Inspector was not auto-scrolling or correctly formatting the Google Vertex JSON payload. Finally, there was a major abstraction leak: `handlers/vein` was executing the `serde_json` serialization and manually formatting the NetworkLog payload, forcing the cognitive center to care about HTTP APIs.
**Resolution:** Applied GTK4 standard pointer piercing (`row.set_can_target(false)`) to correctly trigger the MultiSelection. Leveraged `glib::idle_add_local` with the `vadjustment` to cache and securely anchor the scrolling physics during `matrix_store.splice`. To polish the Adwaita interface, stripped `.hexpand(true)` from the chat bubble `msg_label` to allow dynamic `halign` positioning. Auto-scrolled the network log by appending to the `TextBuffer`, placing a mark at the end iterator, and calling `TextView::scroll_to_mark`. Most crucially, structurally severed the abstraction leak: created a dedicated `format_network_log` module inside `libs/gneiss_pal`, updated `ResilientClient` to accept a cloned `Synapse` event bus, and forced `gneiss_pal` to natively serialize, format, and broadcast its own telemetry. `vein` is now completely agnostic of JSON.
