## 2026-04-03 - [AppKit Hierarchy Anomaly: NSTextView tag rejection]
**Anomaly:** `-[NSTextView setTag:]` throws `NSInvalidArgumentException` because `NSTextView` inherits directly from `NSText` -> `NSView` -> `NSResponder`, bypassing `NSControl`, and thus lacks a default `tag` property. Additionally, the Rust compiler flagged unnecessary `unsafe` blocks around safe `NSLayoutConstraint::activateConstraints` bindings in both `comms.rs` and `sidebar.rs`.
**Resolution:** Excised the `setTag:` / `viewWithTag:` pattern in favor of a fast, zero-copy `NSArray` iteration over `subviews` checking for the `NSTextView` class type. Stripped out the superfluous `unsafe` blocks per the verification harness warnings to achieve absolute compilation silence. To do this, I retained the correct subview pointer explicitly (`Retained::retain(subview as *const _ as *mut _).unwrap()`) before running `Retained::cast_unchecked::<NSTextView>`.
