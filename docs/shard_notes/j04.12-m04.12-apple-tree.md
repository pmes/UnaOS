## 2026-04-03 - [AppKit Hierarchy Anomaly: NSTableView cell collapse]
**Anomaly:** In macOS, an `NSTableView` with `usesAutomaticRowHeights = YES` demands two things from its cell subviews: an unbroken vertical constraint chain, and an intrinsic content size. `NSTextView` does not provide an intrinsic content size to Auto Layout by default. Because the table asks the text view "how tall are you?" and the text view asks the table "how tall are you?", the constraint solver gives up and collapses the row to its absolute minimum height.
**Resolution:** Replaced the `NSTextView` in the chat bubble cell with a multi-line, word-wrapping `NSTextField`. `NSTextField` natively provides its exact text height to the Auto Layout engine, ensuring the `NSTableView` rows dynamically snap to the correct sizes. Lowered horizontal compression resistance (`250.0`) and set cell wrapping (`setWraps: YES`, `setLineBreakMode: 0`). Configured zero-copy stream appending using `stringValue` and `setStringValue:`.
