## 2024-05-24 - [The Purifier: Eradicating set_associated_object Hacks]
**Anomaly:** J04.01 bypassed strict pointer lifecycle semantics by utilizing `objc2::runtime::AnyObject::set_associated_object` with `.as_ptr()` to artificially inflate AppKit delegate retention outside of `AppDelegate` control. Additionally, redundant `unsafe { ... }` blocks were nesting inside safe methods that were structurally verified.
**Resolution:** Ripped out the `AssociationPolicy` hallucinations. Architected a pure `AppDelegateIvars` struct containing `RefCell<Option<Retained<T>>>` references. Altered the cross-boundary `bootstrap_fn` closure to hand over `(NSView, SidebarDelegate, CommsDelegate)` natively. Stripped the unnecessary FFI unsafe block on `split_view.setDelegate()`, respecting the latest stable `objc2` v0.6.4 traits.
