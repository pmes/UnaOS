## 2024-05-24 - J53 Synapse Context Telemetry Implementation
**Anomaly:** The telemetry UI for contextual payload needed to correctly ingest and sync `Vec<WeightedSkeleton>` from the `cortex.rs` async loop into the `vein/src/lib.rs` `State` struct without creating deadlocks across the Tokio event runtime boundaries, while avoiding mutating the universal `SMessage` enum. The `apps/lumen/src/main.rs` compiler error correctly flagged a missing channel configuration when bridging `spline.bootstrap`.
**Resolution:** Employed the `Can-Am` approach by leveraging `async_channel` internally within `lumen` to broadcast the payload to `ContextView`. Updated the `run_indexer` signature in `handlers/vein/src/cortex.rs` to clone the payload for async broadcast while correctly returning the raw `Vec<WeightedSkeleton>` back to `vein/src/lib.rs`. Locked the returned payload momentarily inside a detached `tokio::spawn` task so the UI and `PreFlightPayload` generation have zero-copy, instantaneous access to `live_context`.
